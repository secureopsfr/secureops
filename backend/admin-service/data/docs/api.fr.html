API SecureOps

Ce guide explique comment utiliser l'API publique SecureOps pour automatiser vos scans dans vos pipelines CI/CD, scripts et outils internes.

Vous trouverez ici les endpoints, les paramètres, les exemples curl et des workflows GitHub Actions prêts à l'emploi.

1. Quick Start

Exemple minimal pour créer un scan asynchrone:

curl -X POST "https://api.secureops.fr/scan/api/scan/async" \
	-H "Content-Type: application/json" \
	-H "X-API-Key: sk_votre_cle" \
	-d '{"url":"https://example.com","scan_type":"frontend","input":{"lang":"en"}}'

Réponse attendue:

{
	"job_id": "54738dd7-540e-4288-a367-2951f1a34561",
	"status": "pending",
	"scan_type": "frontend",
	"scan_mode": "passive",
	"job_token": null
}

2. Authentification

Deux modes d'authentification sont supportés :

  • X-API-Key (recommandé en CI/CD)
  • Authorization: Bearer ... (JWT ou clé API selon contexte)

Exemple header recommandé :

-H "X-API-Key: sk_votre_cle"

3. Endpoints API

Méthode Endpoint Description
POST /scan/api/scan/async Crée un job de scan asynchrone single URL
POST /scan/api/scan/multi-async Crée un job multi URL (même domaine)
GET /scan/api/scan/async/{job_id} Retourne le statut du job
GET /scan/api/scan/async/{job_id}/result Retourne le résultat final

4. Scan async single URL

Requête

POST /scan/api/scan/async
Content-Type: application/json
X-API-Key: sk_votre_cle

{
	"url": "https://example.com",
	"scan_type": "frontend",
	"scan_mode": "passive",
	"input": {
		"lang": "en"
	}
}

Paramètres

  • url : URL cible à scanner
  • scan_type : frontend ou backend
  • scan_mode : passive, intrusive, destructive, custom
  • input.lang : fr ou en pour les libellés de findings

Polling statut

curl -H "X-API-Key: sk_votre_cle" \
	"https://api.secureops.fr/scan/api/scan/async/JOB_ID"

Récupération résultat

curl -H "X-API-Key: sk_votre_cle" \
	"https://api.secureops.fr/scan/api/scan/async/JOB_ID/result"

5. Scan async multi URL

Le multi scan permet de scanner plusieurs URLs dans un seul job.

Contrainte importante : toutes les URLs doivent appartenir au même domaine.

Requête

POST /scan/api/scan/multi-async
Content-Type: application/json
X-API-Key: sk_votre_cle

{
	"urls": [
		"https://example.com",
		"https://example.com/login",
		"https://example.com/about"
	],
	"scan_type": "frontend",
	"scan_mode": "passive",
	"input": {
		"lang": "en"
	}
}

Résultat

Le payload final contient notamment :

  • score_global : score global agrégé
  • page_results : détails par URL (score + findings)

6. Codes d'erreur fréquents

Code Cause probable Action recommandée
401 Clé API absente/invalide Vérifier X-API-Key
403 Accès refusé (ownership/job token) Vérifier utilisateur ou token de job
404 Endpoint incorrect ou job introuvable Vérifier l'URL API et le job_id
409 Résultat pas encore prêt Continuer le polling de statut
422 Body JSON invalide/manquant Vérifier le payload envoyé

7. Intégration GitHub Actions

Single URL

name: SecureOps Scan

on:
	push:
		branches: [main]

jobs:
	security:
		runs-on: ubuntu-latest
		steps:
			- uses: actions/checkout@v4

			- name: Run SecureOps scan
				id: scan
				uses: secureopsfr/actions-scan@v1
				with:
					url: https://example.com
					api_key: ${{ secrets.SECUREOPS_API_KEY }}
					fail_on_score_below: "70"

			- name: Print score
				run: echo "Score = ${{ steps.scan.outputs.score }}"

Multi URL

name: SecureOps Scan Multi

on:
	workflow_dispatch:

jobs:
	security-multi:
		runs-on: ubuntu-latest
		steps:
			- uses: actions/checkout@v4

			- name: Run SecureOps scan (multi)
				id: scan
				uses: secureopsfr/actions-scan@v1
				with:
					api_key: ${{ secrets.SECUREOPS_API_KEY }}
					urls: |
						https://example.com
						https://example.com/login
						https://example.com/about
					fail_on_score_below: "70"

			- name: Print score
				run: echo "Score = ${{ steps.scan.outputs.score }}"

Alternative input urls en JSON :

urls: '["https://example.com","https://example.com/login","https://example.com/about"]'

8. Bonnes pratiques

  • Stockez toujours la clé API dans un secret CI, jamais en clair.
  • Commencez avec un seuil raisonnable (fail_on_score_below: "70"), puis augmentez progressivement.
  • En multi scan, gardez des URLs du même domaine.
  • Utilisez input.lang: en si vous voulez des findings en anglais.
  • En cas d'erreur 409, continuez le polling au lieu d'échouer immédiatement.
