Découverte des pages

Cette documentation explique comment fonctionne la découverte des pages avant un scan, quand vous souhaitez analyser plusieurs URLs d'un même site. Elle décrit les modes disponibles, ce qui peut bloquer ou ralentir le processus, et comment interpréter les résultats.

1. À quoi sert la découverte des pages ?

Quand vous lancez un scan sur un site, vous pouvez :

  • Scanner une seule page : vous cochez « Scanner uniquement cette page » et le scan s'exécute directement sur l'URL saisie. Aucune découverte.
  • Découvrir puis scanner : vous laissez la case décochée. Un outil de découverte parcourt votre site pour trouver les autres pages, vous affiche la liste, et vous pouvez la modifier (ajouter, supprimer des URLs) avant de lancer le scan.

La découverte vous permet donc de visualiser la surface du site — quelles pages existent, quels liens sont accessibles — et de choisir ce que vous voulez inclure dans l'analyse. C'est particulièrement utile pour les sites avec beaucoup de pages (blog, e-commerce, portail).

Important : la découverte ne lance pas de scan. Elle ne fait que lister les URLs. C'est vous qui validez la liste et déclenchez le scan ensuite.

2. Quand utiliser la découverte (ou non)

Situation Recommandation
Vous voulez tester une page précise (ex. formulaire de connexion) Cochez « Scanner uniquement cette page »
Votre site a peu de pages, ou vous connaissez déjà les URLs Cochez « Scanner uniquement cette page », ou utilisez la découverte pour vérifier la liste
Votre site a beaucoup de pages (blog, catalogue) et vous voulez une vue d'ensemble Utilisez la découverte pour explorer, puis choisissez les URLs à inclure
Votre site est une SPA (Next.js, React, Vue) avec des liens chargés en JavaScript Utilisez la découverte en mode « Site SPA » pour voir les liens rendus

3. Les modes de découverte

Trois modes sont disponibles selon le type de site :

Mode Comment ça marche Quand l'utiliser
HTML (par défaut) Lit le code HTML de chaque page et extrait les liens. Rapide, peu consommateur. Sites classiques (HTML statique, PHP, templates serveur). Les liens sont présents dans le HTML dès le chargement.
Site SPA Ouvre chaque page dans un navigateur headless, attend que le JavaScript s'exécute, puis extrait les liens du DOM. Plus lent, mais voit les liens générés dynamiquement. Sites Next.js, React, Vue, Angular, etc. Les liens sont injectés par JavaScript après le chargement.
Les deux Exécute HTML et SPA en parallèle, fusionne les résultats. Découverte maximale. Quand vous n'êtes pas sûr du type de site, ou pour une exploration complète. Plus long.

Ce qui peut être bloquant selon le mode

  • Mode HTML sur un SPA : seuls les liens déjà présents dans le HTML initial seront trouvés. Les liens de navigation, articles de blog, etc. rendus par React/Next.js ne seront pas vus. Utilisez le mode SPA.
  • Mode SPA : nécessite plus de ressources (navigateur headless). Si le serveur est surchargé ou si Playwright n'est pas disponible côté infrastructure, le mode peut échouer. En cas d'erreur, testez le mode HTML.

4. Parcours utilisateur étape par étape

  1. Saisir l'URL de la page d'accueil (ou de la page de départ) du site à analyser.
  2. Choisir : « Scanner uniquement cette page » (pas de découverte) ou laisser décoché pour lancer la découverte.
  3. Si découverte : cocher « Site SPA » si votre site est en React/Next.js/Vue, sinon laisser décoché (mode HTML).
  4. Régler la limite d'URLs : nombre maximum de pages à découvrir (par défaut 50, entre 5 et 200).
  5. Lancer : la découverte démarre. Un écran de progression affiche les étapes (validation de l'URL, récupération de robots.txt, sitemap, exploration des pages).
  6. Résultat : vous voyez la liste des URLs trouvées. Vous pouvez supprimer celles que vous ne voulez pas analyser, ou en ajouter manuellement (ex. une page connue qui n'a pas été découverte).
  7. Valider et scanner : cliquez sur « Lancer le scan » pour démarrer l'analyse sur la liste validée.

Note : pour l'instant, le scan s'exécute sur l'URL de départ uniquement. La liste validée est conservée pour une évolution future (scan multi-URLs).

5. Comprendre les résultats

Ce qui est affiché

  • Liste d'URLs : chaque URL trouvée, avec son type (page, formulaire, script, endpoint API, sitemap) et sa profondeur (niveau de clics depuis la page d'accueil).
  • Indicateurs :
    • Délai dépassé : la découverte a été interrompue car le temps imparti était atteint. Les URLs trouvées jusqu'à ce moment sont affichées.
    • Requêtes bloquées : le site a renvoyé plusieurs erreurs 403 (accès refusé), probablement à cause d'une protection anti-bot ou d'un WAF. La découverte s'est arrêtée, mais les URLs déjà trouvées (et celles du sitemap en attente) sont retournées.
    • Protection anti-bot détectée : le contenu de la page suggère une protection (Cloudflare, captcha, etc.). La liste peut être incomplète.

Types d'URLs

  • Page : page HTML classique, lien cliqué par un utilisateur.
  • Formulaire : action d'un formulaire.
  • Script / feuille de style : ressources chargées par la page.
  • Endpoint API : URL qui ressemble à une API (ex. /api/, /graphql/).
  • Sitemap : URL trouvée dans le sitemap (liste de pages fournie par le site pour les moteurs de recherche).

6. Ce qui peut bloquer ou ralentir

Bloquant (la découverte échoue ou s'arrête)

  • URL invalide : format incorrect, domaine inexistant, schéma non supporté (doit être http ou https). Vérifiez l'URL.
  • Site inaccessible : timeout, firewall, site hors ligne. Le scan ne peut pas joindre le serveur.
  • Erreurs 403 répétées : si le site renvoie « Accès refusé » sur plusieurs requêtes consécutives (protection anti-bot, WAF), la découverte s'arrête. Les URLs déjà trouvées sont affichées, mais la liste peut être partielle.
  • Mode SPA indisponible : si l'infrastructure ne supporte pas le mode Playwright (navigateur headless), le mode SPA peut échouer. Utilisez le mode HTML.

Ralentissant (résultats partiels)

  • Délai dépassé : la découverte a une limite de temps (environ 60 à 90 secondes). Les sites très grands ou lents peuvent ne pas être entièrement explorés. Réduisez la limite d'URLs ou relancez plusieurs fois en partant de différentes pages.
  • Profondeur limitée : seuls les liens à 2 niveaux de profondeur sont suivis (accueil → page → sous-page). Des pages plus profondes ne seront pas découvertes automatiquement. Ajoutez-les manuellement si nécessaire.
  • Liens exclus : certains types de liens ne sont pas suivis (images, PDF, archives, etc.) pour éviter de surcharger. C'est normal.

Informatif

  • robots.txt : la découverte respecte le fichier robots.txt du site. Les chemins marqués « Disallow » ne sont pas explorés. Si votre /admin/ est en Disallow, il n'apparaîtra pas dans la liste. Pour l'analyser, ajoutez l'URL manuellement.

7. Limites et cas particuliers

  • Nombre max d'URLs : entre 5 et 200 selon votre réglage (défaut 50). Au-delà, la découverte s'arrête.
  • Profondeur : 2 niveaux. Une page accessible uniquement par 4 clics depuis l'accueil ne sera pas trouvée automatiquement.
  • Même domaine : seuls les liens du même domaine (et sous-domaines) sont suivis. Un lien vers un autre site n'est pas exploré.
  • Sites avec authentification : la découverte ne peut pas se connecter. Seules les pages accessibles sans login seront trouvées.
  • Contenu derrière JavaScript : en mode HTML, les liens chargés dynamiquement ne sont pas vus. Passez en mode SPA.
  • Scan multi-URLs : actuellement, le scan n'analyse que l'URL de départ. La liste validée sert à préparer une future évolution.

8. Conseils pour de meilleurs résultats

  • Commencez par la page d'accueil : c'est le point d'entrée logique. La découverte part de cette URL et suit les liens.
  • Vérifiez le sitemap : si votre site a un sitemap.xml, la découverte le récupère et ajoute ses URLs. Cela améliore la couverture.
  • Utilisez le mode SPA pour les applications modernes : Next.js, React, Vue, etc. Sans ça, la majorité des liens peut être manquante.
  • En cas de requêtes bloquées : si vous voyez « Requêtes bloquées », votre site ou votre hébergeur limite probablement les requêtes automatisées. Vérifiez les règles WAF, les restrictions par IP, ou les protections anti-bot. Vous pouvez toujours ajouter les URLs manuellement.
  • Ajout manuel : si une page importante n'apparaît pas (protégée par robots.txt, trop profonde, ou lien atypique), ajoutez-la à la main dans la liste avant de lancer le scan.
