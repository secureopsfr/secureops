# Vérifications actives — Object storage exposure

Ce document décrit les vérifications actives de lecture non autorisee sur buckets/object storage.

---

## Contexte et méthodologie

### Objectif

Detecter l'exposition de ressources de stockage via patterns d'URL publiques.

### Principe

- Tests de lecture sur objets potentiellement sensibles.
- Verification ACL/public access.
- Aucun test d'ecriture destructive.

## 1. Lecture non autorisée d'objets

### Exemple

- **OK** : acces refuse aux objets non publics.
- **Finding** : lecture non autorisee possible via URL devinee.

### Vulnérabilité et impact

- **Vraisemblance** : Moyenne.
- **Impact** : Importante.
