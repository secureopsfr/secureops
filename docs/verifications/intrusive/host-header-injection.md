# Vérifications actives — Host header injection

Ce document décrit les vérifications actives de Host Header Injection.

---

## Contexte et méthodologie

### Objectif

Detecter les applications qui reutilisent `Host`/`X-Forwarded-Host` sans validation pour generer des URLs absolues, emails ou redirections.

### Tests

- Modifier `Host` et `X-Forwarded-Host`.
- Observer liens de reset password, redirects, canonical URLs.

---

## 1. Injection via Host

### Exemple

- **OK** : host non attendu ignore/rejete.
- **Finding** : lien absolu genere avec host injecte.

### Vulnérabilité et impact

- **Vraisemblance** : Moyenne.
- **Impact** : Significative a importante.

### Conseils

- Fixer un host canonical cote serveur.
- Ignorer headers proxy non fiables sans trust boundary.
- Valider strictement les hosts autorises.
