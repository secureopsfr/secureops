# Vérifications actives — GraphQL subscriptions abuse

Ce document décrit les vérifications actives d'abus sur subscriptions GraphQL (flux temps reel).

---

## Périmètre selon scan_type

| scan_type | Comportement |
|-----------|--------------|
| `frontend` | ✗ **Skip total** — GraphQL subscriptions sont un endpoint backend |
| `backend` | ✓ **Check complet** — si endpoint GraphQL avec subscriptions détecté |

---

## Contexte et méthodologie

### Objectif

Verifier qu'un client ne peut pas s'abonner a des flux sensibles hors perimetre.

### Principe

- Test d'abonnement a des topics non autorises.
- Verification scope tenant/utilisateur.
- Rejeu avec token expire/revoque.

## 1. Souscriptions non autorisées

### Exemple

- **OK** : abonnement non autorise refuse.
- **Finding** : fuite de flux cross-tenant.

### Vulnérabilité et impact

- **Vraisemblance** : Moyenne.
- **Impact** : Importante.
