# Vérifications actives — Service mesh / internal API exposure

Ce document décrit les vérifications actives d'exposition accidentelle d'APIs internes.

---

## Contexte et méthodologie

### Objectif

Identifier des routes internes devenues atteignables depuis l'exterieur par erreur de routage/proxy.

### Principe

- Detection d'endpoints internes courants.
- Verification auth/authz et segmentation reseau.
- Correlation avec config proxy/gateway.

## 1. Exposition d'API interne

### Exemple

- **OK** : endpoints internes inaccessibles publiquement.
- **Finding** : route interne sensible atteignable sans controles adequats.

### Vulnérabilité et impact

- **Vraisemblance** : Faible a moyenne.
- **Impact** : Importante.
