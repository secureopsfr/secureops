# Vérifications actives — SSRF applicative

Ce document décrit les vérifications actives SSRF applicative (parametres URL et parsing).

---

## Contexte et méthodologie

### Objectif

Detecter les endpoints qui recuperent des ressources distantes sans filtrage robuste des destinations.

### Principe

- Tests sur parametres de type URL/webhook/fetch.
- Variantes de parsing (scheme, encodages, redirections).
- Mode safe : pas d'exfiltration, preuves minimales.

---

## 1. SSRF via parametres URL

### Exemple

- **OK** : destinations privees/interdites bloquees.
- **Finding** : endpoint tente d'atteindre une cible hors scope autorise.

### Vulnérabilité et impact

- **Vraisemblance** : Moyenne.
- **Impact** : Importante a majeure.

### Conseils

- Allowlist stricte des hosts/schemes.
- Resolution DNS robuste + blocage IP privees/loopback.
- Verification apres redirection.
